# Labels recommandés

## Labels minimum
- `documentation`
- `enhancement`
- `bug`
- `ci`
- `tests`
- `architecture`
- `objective-2`

## À quoi servent les labels
Ils servent à classer les issues par nature :
- doc,
- amélioration,
- bug,
- infrastructure,
- qualité.

## Remarque
Tu as déjà au moins `documentation` et `enhancement` visibles sur tes issues. Tu peux donc garder cette logique et l’élargir.
