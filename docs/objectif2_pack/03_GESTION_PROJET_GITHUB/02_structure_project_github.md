# Structure du Project GitHub

## Colonnes conseillées
- Backlog
- Ready
- In progress
- In review
- Done

## Utilisation
- Backlog : idées non commencées
- Ready : prêtes à être prises
- In progress : en cours
- In review : en attente de vérification
- Done : terminé

## Ce qui existe déjà
Ton Project suit déjà cette logique, ce qui est très bien pour l’objectif 2.
Le but n’est donc pas de refaire le board, mais de l’utiliser comme preuve.
